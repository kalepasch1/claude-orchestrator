## Key Concepts

### Task Execution Flow
```
1. User queues task from web dashboard (or API)
   └─ Task: prompt + project_id + deps
2. Runner polls Supabase, claims task → RUNNING
3. Runner creates git worktree for project
4. Runner executes: CLAUDE_BIN=claude python3 runner.py
   └─ Claude Code runs in isolation, reports back
5. Verify step: cheap model reviews diff
   └─ If risky → approval card → web dashboard
   └─ If safe → integrate (git commit + push)
6. Outcome recorded: cost ($), tokens, status
7. Dashboard reflects live changes in real-time
```

### Multi-Account Rotation
- `account_pool.py` cycles across your **authorized Anthropic accounts**
- Routes by usage: picks account with lowest usage when exhausted
- ⚠️ **SAFETY:** Only use accounts you're entitled to. Stay within Anthropic's usage policies.
- Model router heuristic (cost vs. throughput) when cold; bandit learning kicks in after ~100 runs

### Knowledge Reuse
- `knowledge.py` + pgvector: semantic embedding of prior outcomes
- New task → embed prompt → find similar past solutions
- Cached context prefix (stable 16k-32k token block) reduces input cost

### Self-Improvement Loop
- `eval_harness.py` reads telemetry (token spend, failure patterns)
- Proposes orchestrator prompt improvements as **approval cards**
- Never self-edits silently — all material changes via git + CI + your approval
- A/B gates prompt changes before adoption

## DO / DON'T

**DO:**
- Approve material changes (all self-edits, multi-project diffs) before merge
- Test execution with `--dry-run` before running real task
- Set project budgets in `budgets` table (prevents runaway spend)
- Rotate accounts fairly — don't exhaust one while others idle
- Use contract-first DAGs (task 1 = shared types/API, deps lock in contract early)
- Review `verify.py` diffs carefully before approving (risky = needs review)
- Schedule runner via launchd (plist) for durability — reuse `scripts/setup-scheduler.sh`
- Keep `runner/.env` out of git — add to `.gitignore` if missing
- Handle transient errors (HTTP 409) in central location; treat as requeues not terminal
- Implement fallbacks for missing/malformed data (use best-effort ranking)
- Trust worktree_gc: it only removes stale worktrees from completed tasks (never RUNNING/RETRY tasks)

**DON'T:**
- Commit `runner/.env`, `web/.env`, or any `SUPABASE_SERVICE_KEY` to git
- Use unauthorized Anthropic accounts (violates usage policy)
- Skip approval cards — self-changes must go through git + CI + your approval
- Run raw SQL via shell; use Supabase migrations for schema changes
- Hardcode project paths — query `projects` table for repo_path
- Exhaust one account while leaving others idle — bandit learns, but rotate fairly
- Assume outcomes cached forever — re-embed knowledge on schema updates
- Bypass RLS policies with direct untyped queries
- Rely on specific error codes (409, etc.) — handle HTTP errors by behavior (transient vs. terminal)
- Hardcode error messages — use generic error handling approach

#### # project conventions (CLAUDE.md):  
#### # Claude Orchestrator — Multi-Project Task Executor  
####   
#### **Tech Stack:** Nuxt 3 (Vue 3 SSR) + Supabase + Vercel (web) · Python 3.11+ (runner) · Anthropic API v1 · pgvector + Postgres  
####   
#### **Mission:** Distributed execution of Claude Code across all your projects. Auto-route tasks by cost/throughput via bandit learning. All changes require human approval before merging.  
####   
#### ## Directory Map  
####   
#### | Path | Purpose |  
#### |------|---------|  
#### | `web/` | Nuxt 3 SPA dashboard (Vercel-hosted) |  
#### | `runner/` | Python task executor (runs on your Mac) |  
#### | `supabase/` | Migrations, RLS policies, edge functions |  
#### | `runner/runner.py` | Main loop: poll → claim → execute → verify → integrate |  
#### | `runner/bandit.py` | Model selection via UCB1 (cost/throughput) |  
#### | `runner/account_pool.py` | Rotate API keys across **authorized** accounts |  
#### | `runner/caching.py` | Stable cached prefix (token savings) |  
#### | `runner/knowledge.py` | pgvector semantic search + embeddings |  
#### | `runner/verify.py` | Cheap-model review before integration |  
#### | `runner/eval_harness.py` | Self-improvement: propose, test, gate, approve |  
#### | `supabase/functions/` | Slack notifications, approvals webhook |  
####   
#### ## Key Concepts  
####   
#### ### Task Execution Flow  
#### ```  
#### 1. User queues task from web dashboard (or API)  
####    └─ Task: prompt + project_id + deps  
#### 2. Runner polls Supabase, claims task → RUNNING  
#### 3. Runner creates git worktree for project  
#### 4. Runner executes: CLAUDE_BIN=claude python3 runner.py  
####    └─ Claude Code runs in isolation, reports back  
#### 5. Verify step: cheap model reviews diff  
####    └─ If risky → approval card → web dashboard  
####    └─ If safe → integrate (git commit + push)  
#### 6. Outcome recorded: cost ($), tokens, status  
#### 7. Dashboard reflects live changes in real-time  
#### ```  
####   
#### ### Multi-Account Rotation  
#### - `account_pool.py` cycles across your **authorized Anthropic accounts**  
#### - Routes by usage: picks account with lowest usage when exhausted  
#### - ⚠️ **SAFETY:** Only use accounts you're entitled to. Stay within Anthropic's usage policies.  
#### - Model router heuristic (cost vs. throughput) when cold; bandit learning kicks in after ~100 runs  
####   
#### ### Knowledge Reuse  
#### - `knowledge.py` + pgvector: semantic embedding of prior outcomes  
#### - New task → embed prompt → find similar past solutions  
#### - Cached context prefix (stable 16k-32k token block) reduces input cost  
####   
#### ### Self-Improvement Loop  
#### - `eval_harness.py` reads telemetry (token spend, failure patterns)  
#### - Proposes orchestrator prompt improvements as **approval cards**  
#### - Never self-edits silently — all material changes v

## Key Concepts

### Task Execution Flow
```
1. User queues task from web dashboard (or API)
   └─ Task: prompt + project_id + deps
2. Runner polls Supabase, claims task → RUNNING
3. Runner creates git worktree for project
4. Runner executes: CLAUDE_BIN=claude python3 runner.py
   └─ Claude Code runs in isolation, reports back
5. Verify step: cheap model reviews diff
   └─ If risky → approval card → web dashboard
   └─ If safe → integrate (git commit + push)
6. Outcome recorded: cost ($), tokens, status
7. Dashboard reflects live changes in real-time
```

"""
Patch template rendering utilities for adapting patches to target environments.
"""

import re
from typing import Dict, Any

def render_patch_template(
    patch_content: str,
    context: Dict[str, Any]
) -> str:
    """
    Render a patch template with provided context.
    
    Args:
        patch_content: Raw patch content that may contain template placeholders
        context: Context dictionary with values to substitute
        
    Returns:
        Rendered patch content
    """
    # Handle common template patterns in the patch
    rendered_content = patch_content
    
    # Replace branch name placeholders
    if 'branch_name' in context:
        rendered_content = re.sub(
            r'\b(?:origin/)?(main|master|develop)\b',
            context['branch_name'],
            rendered_content
        )
    
    # Replace commit hash placeholders (if any)
    if 'commit_hash' in context:
        rendered_content = re.sub(
            r'\b[0-9a-f]{7,40}\b',
            context['commit_hash'],
            rendered_content
        )
    
    # Replace file path placeholders
    if 'file_path' in context:
        rendered_content = re.sub(
            r'\b(?:src/|lib/)?[^/\s]+\.(?:js|ts|py|java|cpp)\b',
            context['file_path'],
            rendered_content
        )
    
    # Replace version placeholders
    if 'version' in context:
        rendered_content = re.sub(
            r'\b(?:v\d+\.\d+\.\d+)\b',
            context['version'],
            rendered_content
        )
    
    # Handle any other template-specific patterns
    for key, value in context.items():
        placeholder = f"{{{key}}}"
        rendered_content = rendered_content.replace(placeholder, str(value))
    
    return rendered_content

def extract_template_context(patch_content: str) -> Dict[str, Any]:
    """
    Extract context information from a patch template.
    
    Args:
        patch_content: Raw patch content
        
    Returns:
        Dictionary with extracted context information
    """
    context = {}
    
    # Extract branch names
    branch_patterns = [
        r'\b(?:origin/)?(main|master|develop)\b',
        r'branch:\s*(\w+)',
        r'refs/heads/(\w+)'
    ]
    
    for pattern in branch_patterns:
        matches = re.findall(pattern, patch_content, re.IGNORECASE)
        if matches:
            context['branch_name'] = matches[0]
            break
    
    # Extract commit hashes
    commit_pattern = r'\b([0-9a-f]{7,40})\b'
    commit_matches = re.findall(commit_pattern, patch_content)
    if commit_matches:
        context['commit_hash'] = commit_matches[0]
    
    # Extract version information
    version_pattern = r'\b(v\d+\.\d+\.\d+)\b'
    version_matches = re.findall(version_pattern, patch_content)
    if version_matches:
        context['version'] = version_matches[0]
    
    return context

def adapt_patch_for_target(
    patch_content: str,
    target_environment: Dict[str, Any]
) -> str:
    """
    Adapt a patch for a specific target environment.
    
    Args:
        patch_content: Raw patch content
        target_environment: Target environment information
        
    Returns:
        Adapted patch content
    """
    # Render the template with target context
    adapted_content = render_patch_template(patch_content, target_environment)
    
    # Apply any additional adaptations based on target environment
    if target_environment.get('target_branch'):
        # Update branch references in the patch
        adapted_content = re.sub(
            r'\b(?:origin/)?(main|master|develop)\b',
            target_environment['target_branch'],
            adapted_content
        )
    
    return adapted_content

    Args:
        patch_content: The raw patch content to transplant
        target_codebase: The target codebase content to apply the patch against
        common_ancestor: Optional common ancestor for three-way merge

                common_ancestor

        old_content: Original content
        new_content: New content
        context_lines: Number of context lines to include
        
    Returns:
        Unified diff string
    """

    old_lines = old_content.splitlines(keepends=True)
    new_lines = new_content.splitlines(keepends=True)
    
    # Generate unified diff
    diff = difflib.unified_diff(
        old_lines,
        new_lines,
        fromfile='a',
        tofile='b',
        lineterm='',
        n=context_lines
    )
    
    return ''.join(diff)

def _diff_contents(
    content_a: str,
    content_b: str
) -> List[Tuple[str, int, int]]:
    """
    Compare two content strings and return differences.
    
    Args:
        content_a: First content
        content_b: Second content
        
    Returns:
        List of differences with their positions
    """
    # Split into lines
    lines_a = content_a.splitlines(keepends=True)
    lines_b = content_b.splitlines(keepends=True)
    
    # Use difflib to get the differences
    differ = difflib.SequenceMatcher(None, lines_a, lines_b)
    
    differences = []
    for tag, i1, i2, j1, j2 in differ.get_opcodes():
        if tag != 'equal':
            differences.append((tag, i1, i2))
    
    return differences

"""

import re
from typing import Dict, Any

def render_patch_template(
    patch_content: str,
    context: Dict[str, Any]
) -> str:
    """
    Render a patch template with provided context.
    
    Args:
        patch_content: Raw patch content that may contain template placeholders
        context: Context dictionary with values to substitute
        
    Returns:
        Rendered patch content
    """
    # Handle common template patterns in the patch
    rendered_content = patch_content
    
    # Replace branch name placeholders
    if 'branch_name' in context:
        rendered_content = re.sub(
            r'\b(?:origin/)?(main|master|develop)\b',
            context['branch_name'],
            rendered_content
        )
    
    # Replace commit hash placeholders (if any)
    if 'commit_hash' in context:
        rendered_content = re.sub(
            r'\b[0-9a-f]{7,40}\b',
            context['commit_hash'],
            rendered_content
        )
    
    # Replace file path placeholders
    if 'file_path' in context:
        rendered_content = re.sub(
            r'\b(?:src/|lib/)?[^/\s]+\.(?:js|ts|py|java|cpp)\b',
            context['file_path'],
            rendered_content
        )
    
    # Replace version placeholders
    if 'version' in context:
        rendered_content = re.sub(
            r'\b(?:v\d+\.\d+\.\d+)\b',
            context['version'],
            rendered_content
        )
    
    # Handle any other template-specific patterns
    for key, value in context.items():
        placeholder = f"{{{key}}}"
        rendered_content = rendered_content.replace(placeholder, str(value))
    
    return rendered_content

def extract_template_context(patch_content: str) -> Dict[str, Any]:
    """
    Extract context information from a patch template.
    
    Args:
        patch_content: Raw patch content
        
    Returns:
        Dictionary with extracted context information
    """

    branch_patterns = [
        r'\b(?:origin/)?(main|master|develop)\b',
        r'branch:\s*(\w+)',
        r'refs/heads/(\w+)'
    ]
    
    for pattern in branch_patterns:
        matches = re.findall(pattern, patch_content, re.IGNORECASE)
        if matches:
            context['branch_name'] = matches[0]
            break
    
    # Extract commit hashes
    commit_pattern = r'\b([0-9a-f]{7,40})\b'
    commit_matches = re.findall(commit_pattern, patch_content)
    if commit_matches:
        context['commit_hash'] = commit_matches[0]
    
    # Extract version information
    version_pattern = r'\b(v\d+\.\d+\.\d+)\b'
    version_matches = re.findall(version_pattern, patch_content)
    if version_matches:
        context['version'] = version_matches[0]
    
    return context
