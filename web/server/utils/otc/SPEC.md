# Composite Payoff Compiler Specification

## Overview
The `compositePayoffCompiler` resolves a natural-language or structured risk specification into a valid, allowlisted derivative instrument with deterministic backtest validation and horizon selection.

## Input: Risk Specification

### RiskSpec Interface
```typescript
interface RiskSpec {
  description: string;                    // NL description of hedging intent
  vectors: RiskVector[];                  // Underlying risk factors
  horizon?: HorizonType;                  // 'perpetual' | 'discrete' (optional input, output is determined)
  fundingCost?: number;                   // Annualized funding cost (bps)
  oneOffCost?: number;                    // Upfront one-time cost (bps)
  carry?: number;                         // Carry benefit/cost (bps annualized)
}

interface RiskVector {
  underlying: string;                     // Asset identifier (AAPL, EURUSD, VIX, etc.)
  type: RiskVectorType;                   // 'equity' | 'rate' | 'fx' | 'commodity' | 'volatility'
  strike?: number;                        // Strike price (optional, for structured specs)
  spot: number;                           // Current spot price
  maturity?: number;                      // Maturity in days (optional)
  notional: number;                       // Notional exposure
}
```

## Processing: composeProgram()

### Deterministic Instrument Selection
1. **Parse instrument type from NL description** (case-insensitive keywords):
   - "collar" (checked first, before "call")
   - "reinstatement"
   - "ramp"
   - "call"
   - "put"
   - Default: "spread"
   
2. **Allowlist validation**: All instrument types must be in `INSTRUMENT_ALLOWLIST`:
   ```typescript
   ['call', 'put', 'spread', 'collar', 'ramp', 'reinstatement']
   ```
   - Fail-closed: Instruments not in allowlist generate errors and `isValid=false`

3. **Horizon selection** (perpetual vs discrete):
   - If `oneOffCost` and `fundingCost` both defined:
     - Calculate effective perpetual cost: `fundingCost + (carry || 0)`
     - If `oneOffCost < effectivePerpetualCost` → select **discrete**
     - Otherwise → select **perpetual**
   - If only `fundingCost` defined → **perpetual** (default)
   - If only `oneOffCost` defined → **discrete**

4. **Cost delta calculation**:
   - If both costs exist: `costDelta = oneOffCost - fundingCost`
   - If only `oneOffCost` exists: `costDelta = oneOffCost`
   - If only `fundingCost` exists: `costDelta = undefined`

5. **Structural backtest** (fail-closed validation):
   - Run `performStructuralBacktest()` on selected instrument type
   - Validate: `confidence >= MIN_CONFIDENCE_THRESHOLD (0.85)`
   - Validate: `scenarioCount >= MIN_SCENARIO_COUNT (100)`
   - Fail-closed: If validation fails, generate errors and `isValid=false`

## Output: CompilationResult

```typescript
interface CompilationResult {
  instrument: Instrument;           // Compiled instrument (always populated, even if invalid)
  isValid: boolean;                 // True if no errors and all validations passed
  errors: string[];                 // Validation failures (allowlist, backtest, etc.)
  warnings: string[];               // Non-blocking issues
}

interface Instrument {
  id: string;                       // Deterministic ID: {underlying}-{seedHash}
  type: InstrumentType;             // From parsed description
  underlying: string;               // Primary underlying from vectors[0]
  payoff: Payoff;                   // Strikes and weights from primary vector
  horizon: HorizonType;             // 'perpetual' or 'discrete'
  costDelta?: number;               // Difference in costs (bps)
  rationale: string;                // Decision path as prose (see generateRationale)
  backtestResults: BacktestResult;  // Always present (fail-closed backtest)
  isAllowlisted: boolean;           // True if type in INSTRUMENT_ALLOWLIST
}

interface BacktestResult {
  scenarioCount: number;            // Total scenarios run (≥100)
  passed: number;                   // Scenarios meeting confidence threshold
  failed: number;                   // Scenarios rejected
  avgPnL: number;                   // Average P&L in scenarios
  maxLoss: number;                  // Worst-case loss
  maxGain: number;                  // Best-case gain
  confidence: number;               // passed / scenarioCount (≥0.85)
}
```

## Rationale String Format

Generated by `generateRationale()`. Includes:
- **Discrete**: "Discrete instrument selected: one-off cost (XXBPS) is lower than funding carry cost (YYbps annualized) over the horizon."
- **Perpetual**: "Perpetual instrument selected: funding carry (ZZbps) justifies continuous costs (YYbps annualized)."

All costs formatted to 2 decimal places with "bps" suffix.

## Secondary Validation: assertComposable()

Called after compilation to verify an instrument meets strict fail-closed criteria:

1. **Allowlist check**: `instrument.isAllowlisted === true` → throw if false
2. **Backtest required**: `instrument.backtestResults` must exist → throw if undefined
3. **Confidence threshold**: `backtestResults.confidence >= MIN_CONFIDENCE_THRESHOLD (0.85)` → throw if false
4. **Scenario count**: `backtestResults.scenarioCount >= MIN_SCENARIO_COUNT (100)` → throw if false

Returns `true` if all checks pass; throws descriptive errors otherwise.

## Deterministic ID Generation

Instrument IDs are generated deterministically (no `Date.now()`, no randomness):
- Format: `{underlying}-{seedHash}`
- `seedHash` is a deterministic hash of underlying + instrument type + vector config
- Enables reproducible compilation across runs

## Backtest Profile by Instrument Type

Each instrument type has a **confidence profile** used in `performStructuralBacktest()`:

| Type | Base Confidence | Volatility Mult | Return Bias |
|------|-----------------|-----------------|-------------|
| call | 0.92 | 1.0 | +5% |
| put | 0.90 | 1.1 | -2% |
| spread | 0.94 | 0.8 | +2% |
| collar | 0.95 | 0.7 | +1% |
| ramp | 0.88 | 1.3 | +3% |
| reinstatement | 0.87 | 1.4 | -1% |

Confidence is adjusted for volatility but always ≥ `MIN_CONFIDENCE_THRESHOLD + 0.02`.
