# Apparently — filings-as-code, deadline omniscience, regulator data rooms

Target repo: apparently.
Source specs: REVIEW v2 §2, REVIEW v3 §3–4. Depends on queued D1–D5 (don't re-implement). RLS/default-deny and ai-call-logging conventions per repo CLAUDE.md.

## Objectives

1. **A4v2 Behavioral learning, default-on** [MATERIAL] — scrub-before-learn pipeline (PII/IP/client-identifier strippers ahead of the learning store) with per-event kernel receipts proving dropped fields; ToS disclosure + opt-out toggle; legitimate-interest assessment doc stored in repo; network-dividend surface ("your lanes are N% faster because…"). Proof: test asserts identifiers never reach learning store; receipt verifies.
2. **A6 Deadline Omniscience** — full lifecycle deadline ontology (pre-application windows, intent notices, comment periods, exam windows, expirations, renewal look-backs, fee schedules); invariant "no silent deadline": every dated obligation has owner + T-90/30/7/24h countdown ladder + dead-man switch (unverified completion at T-x → escalate to accountable officer + open Smarter validation matter via bridge). User-visible autonomous ops feed (timeline of done/next with receipts). Completeness engine: submissions fail-closed unless checklist complete or officer-waiver receipt exists. Proof: fixture calendar test proves ladder fires, dead-man escalates, incomplete submission blocked.
3. **A7v3 Filings-as-code + instant obligations** [MATERIAL] — webhook listeners (GitHub/data pipelines/product config); change classifier mapping diffs → affected filing sections; auto-regenerated filing artifacts shown as diffs; CI gate: filing build FAILS if repo state and filing text diverge (G8 eval-gate pattern); event-triggered drafting for threshold events (notional/AUM crossings, new product flags, jurisdiction activation, accountable-personnel changes) → e.g. CFTC exhibits staged within minutes; auto-file only on A2-certified lanes, else officer sign-off; cross-team sign-off chain via Smarter (compliance/eng/legal confirm sections; reminders ride A6 ladders; final-submission receipt bundles all sign-offs, H4 pattern). Proof: fixture merge regenerates exhibit + fails CI on divergence; threshold event drafts artifact in test.
4. **R2 Regulator Data Rooms** [MATERIAL] — per-regulator portal: live compliance dashboard; natural-language query over evidence graph with every answer tracing to signed receipts; examination mode (pre-built binders, sampling, workpaper export); scoped access (jurisdiction-limited) + immutable mutual audit log of regulator queries; correspondence hub (RFIs as SLA-clocked tickets); email fallback preserved. Regulator query patterns feed A1 requirements graph. Proof: access-scope test, query-trace test, audit-log immutability test.
5. **A1 Requirements-graph crawler + diff-watch** — versioned requirements graph per (jurisdiction × sector × license type); nightly diffs → auto-regenerate affected filings + customer alert. Proof: fixture rule change triggers regeneration.
6. **A2 Lane certification via shadow filings** [MATERIAL] — lanes graduate drafted → officer-reviewed → auto-file only after N shadow replays against historical successful filings pass (G9 pattern). Proof: uncertified lane cannot auto-file (fail-closed test).
7. **A3 Cross-license sequencing solver** — dependency DAG over multi-state/multi-sector prerequisites → critical path + optimal filing order. Proof: fixture 3-state scenario yields correct ordering.
8. **A5 Perpetual compliance calendar as retention product** — renewals/CE/annual reports/exam-readiness binders as subscription surface; auto-assembled examiner packets. Proof: binder assembly test.
9. **SC2 Regulator Interaction Playbook** — per-agency/per-contact professional-interaction intelligence: response-time patterns, preferred channels, contact windows, procedural preferences, published positions, docket/busy-season inference from public data; auto-applied to A6 ladders + R2 correspondence scheduling. HARD CONSTRAINT: professional-pattern data only — no personal-life data fields may exist in the schema (enforce via schema test). Proof: send-time optimization test + schema-constraint test.

Cross-app: consume the shared regime-change oracle + connector layer from the portfolio prompt (X2); do not build a second oracle here.
