  prompt: |
    Supabase went down 3x in 24h and each outage froze all claiming/recording. Build the
    failover layer: runner/local_queue.py — a SQLite mirror (.runtime/queue_mirror.db) that
    (a) continuously shadows QUEUED/RUNNING task rows whenever the DB is up (piggyback on the
    existing poll loop, no extra query volume: mirror what claim_task already fetched);
    (b) when db.select/claim raises repeatedly (reuse sentinel's threshold), runner claims from
    the mirror instead — SINGLE-MACHINE only (env ORCH_OFFLINE_CLAIM_HOST allowlists ONE

    matches the mirror's last writer);
    (c) outcomes/state transitions during the outage are journaled append-only
    (.runtime/offline_journal.jsonl) and replayed via upsert on recovery (upsert-only writes are
    already a SPEC invariant, so replay is idempotent; guard transitions: never QUEUED->MERGED);
    (d) the sentinel's on_db_recovery() triggers the replay.
    Unit tests with a fake db module: mirror sync, offline claim, journal replay idempotency,
    double-claim prevention across hosts.
