PROJECT: beethoven

# BOT FACTORY + first wave of 5 expert-bot specs. beethoven = orchestrator (Python
# runner + pytest). The factory turns a BotSpec (sourced-data + corpus_filter +
# priors + eval) into a VALIDATED, calibration-GATED persona manifest compatible with
# the existing `runDetermination(issue, roster, {invoker, embedder})` roster framework
# (see CADE_IMPLEMENTATION_HANDOFF.md). Every bot must PASS its golden eval to be
# admitted — no un-evaluated bot goes live. All specs build + eval with an INJECTED
# invoker (deterministic), so `pytest` stays green with no live model/corpus. Wiring a
# bot into an app's live legal path is a separate human-approved (OPERATOR) step.

    live capability registry. CANDIDATE-SHARED: the factory + manifest shape are the
