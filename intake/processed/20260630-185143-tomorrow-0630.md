- id: term-structure-netting
  title: Maturity-laddered (term-structure) netting across the forward curve
  material: no
  model: opus
  depends: []
  proof: `npx vitest run --config vitest.pure.config.ts --pool=forks server/utils/risk/__tests__/termStructureNetting.test.ts` exits 0
  prompt: |
    Today marginalRisk nets on spot factor magnitude (a single DV01-style sigma per factor). Add a
    time axis so a 5y loan's year-3 rate risk nets against a policy's year-3 liability — CCP-style
    bucketed compression that surfaces offsets invisible to spot netting.
    Steps:
    - New pure module server/utils/risk/termStructureNetting.ts. Extend the FactorExposure concept
      with an optional tenor bucket: define `BucketedExposure { factorKey, riskClass, tenorBucket
      ('0-1y'|'1-3y'|'3-5y'|'5-10y'|'10y+'), sigmaUsd }`. Provide `bucketize(position, cashflowSchedule)`
      that decomposes a position's sigma across buckets, and `netByBucket(positions)` that nets within
      (factorKey, tenorBucket) then aggregates with a tenor-adjacency correlation (adjacent buckets
      ~0.9, far buckets lower).
    - Reuse marginalRisk.portfolioRisk by mapping each (factorKey,tenorBucket) to a composite factor
      key `${factorKey}@${tenorBucket}` and a correlation fn that applies the adjacency matrix. Do NOT
      modify marginalRisk.ts signatures — compose on top.
    - Add a `loanCashflowBuckets(loan)` helper in server/utils/otc/lending/loanRiskMapper.ts (additive
      export) that returns the tenor distribution of a loan's rate sigma (amortizing vs bullet).
    - Tests server/utils/risk/__tests__/termStructureNetting.test.ts: two opposite-signed exposures in
      the SAME bucket net to ~0; the same two in far-apart buckets net much less; bucket sigmas sum
      back to the position's spot sigma (conservation). Pure, no DB/IO.
