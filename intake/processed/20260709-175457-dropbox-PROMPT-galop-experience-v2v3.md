# Galop — engagement stack, Card Mode multi-race betting, paid-play production hardening

Target repo: galop (racefeed + console + supabase).
Source specs: REVIEW v2 §5, REVIEW v3 §5. Depends on queued E1–E3. Check GALOP_* roadmap docs at repo root before decomposition to avoid duplicating queued roadmap items.

## Objectives

1. **GE1 Engagement stack — free AND paid play** — seasons/battle passes themed to racing calendar, daily quests, streaks, leagues/relegation, cosmetic collections + surprise cosmetic drops on login/content/milestone events, social syndicates (G5), limited-time events synced to live race cards, calibration mastery tree (G1) as prestige axis. Runs in both free and paid surfaces. HARD CONSTRAINT (enforce via kernel constitution, E2 pattern): reward randomness must never be coupled to wager placement, size, or frequency — random/surprise rewards key off login, content, streaks, milestones, social events only; wager-triggered rewards must be fixed/earned-schedule. Constitution test required: `wager_coupled_variable_reward` → deny.
2. **G1 Calibration score core loop** — Brier-type forecast-quality scoring vs tote on every pick (real and free-play); skill tree, leagues, badges on calibration improvement; post-race "what the winner had" analysis cards; pre-bet field-context card (pace/bias/class). Proof: scoring engine unit tests on fixture races.
3. **G2 Cold-streak circuit breakers** — rolling drawdown triggers → stake cool-down, free-play routing (free-play events as the landing zone), coaching interstitial, optional self-set loss limits; routed through kernel constitution so enforcement is provable. Proof: fixture streak triggers breaker; receipt minted.
4. **GE2 Card Mode (multi-race velocity)** — (a) swipe-stack of upcoming races across all circuits, one-tap bet from saved presets; (b) Portfolio Slip: unlimited races in one slip, single confirm, consolidated settlement view, collapsed drawer + badge (interface stays clean); (c) Strategy Apply: rule ("$10 win on top calibration pick, every graded stakes today") applied across card with one approval, per-line editable; (d) always-visible aggregate at-risk + daily P&L in drawer (velocity never hides exposure). Proof: slip aggregation tests; UI decision-budget lint.
5. **Paid-play production hardening** [MATERIAL] — vendor integration matrix (licensed ADW/wagering partner links, official tote/price feeds, settlement rails, per-jurisdiction geo/KYC via E1/E2) tracked on a live integration dashboard; contracted video sources per circuit incl. international overnight coverage (JRA/HK/AUS/harness) with multi-source auto-failover and latency budget; 24/7 synthetic golden-journey probes (place-bet, video-start, odds-refresh, settlement) every few minutes across regions/devices with paging + incident receipts + public status page; settlement-time SLO with receipts and DQ/photo explanation cards. Free-play reruns/mystery content never presented as bettable on the paid surface. Proof: probe harness runs against staging; failover sim test; status page renders.
6. **G3 Verified tipster ledger** — pre-race timestamped public picks, tamper-evident ROI ledgers (signed receipts); creator economy rails. Proof: ledger tamper test.
7. **N4 Reliability stack as B2B** — package the probe/failover/status infra for white-label (extends GALOP_WHITELABEL_HANDOFF). Proof: deployable as standalone module.
